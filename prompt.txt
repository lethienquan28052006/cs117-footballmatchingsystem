# Football Matching and Scheduling Optimization

## 1. Mục tiêu app

Xây dựng một single-page webapp có thể demo quy trình:

Input teams
→ Build compatibility graph
→ Calculate score
→ Filter edges
→ Greedy matching
→ Local search improvement
→ Assign court/time slot
→ Evaluation
→ Final schedule

Webapp dùng để trình bày đồ án Computational Thinking CS117, nên cần trực quan, dễ hiểu, chạy ổn định trên local.

## 7. Parameters

Có panel cho người dùng chỉnh:

* lambda: hệ số phạt skill gap, default 0.36
* maxSkillGap: ngưỡng chênh lệch trình độ tối đa, default 3
* matchingFee: phí dịch vụ ghép trận mỗi đội, default 50000
* algorithm mode:

  * Greedy only
  * Greedy + Local Search

Dùng slider hoặc input number.

## 8. Algorithm details

### 8.1 Build graph

Duyệt tất cả cặp đội i, j.

Một cạnh hợp lệ nếu:

```text
commonSlots.length > 0
and
abs(skill_i - skill_j) <= maxSkillGap
```

Tính:

```text
estimatedProfit = pay_i + pay_j + 2 * matchingFee
score = estimatedProfit - lambda * skillGap * 100000
```

Nếu score > 0 thì thêm edge.

### 8.2 Greedy matching

Sort edges giảm dần theo score.

Chọn edge nếu cả hai team chưa được dùng.

Return matching.

### 8.3 Local search

Triển khai local search đơn giản:

* Input: greedy matching, all edges
* Thử cải thiện bằng cách thay hai matched edges bằng hai edges khác không trùng team
* Nếu tổng score tăng thì nhận thay đổi
* Lặp tối đa 50 iterations hoặc đến khi không cải thiện

Không cần quá tối ưu, nhưng phải chạy ổn định.

### 8.4 Scheduling

Sau khi có matching, gán sân và khung giờ:

* Với mỗi match, tìm courtSlot còn trống có time nằm trong commonSlots
* Nếu cả hai đội có preferredCourts thì ưu tiên court nằm trong preferredCourts
* Nếu nhiều slot hợp lệ, chọn slot có courtFee thấp nhất
* Mỗi court-time chỉ dùng một lần
* finalProfit = estimatedProfit - courtFee

Nếu không tìm được slot thì bỏ match khỏi schedule.

### 8.5 Metrics

Tính:

```text
totalProfit = sum(finalProfit)
avgSkillGap = average(skillGap)
matchedTeams = schedule.length * 2
matchRate = matchedTeams / totalTeams
fillRate = schedule.length / totalAvailableCourtSlots
runtimeMs = performance.now() difference
totalMatches = schedule.length
```

Khi người dùng bấm “Generate Pareto Curve”, chạy nhiều giá trị lambda:

```text
[0, 0.1, 0.2, 0.36, 0.5, 0.75, 1.0]
```

Với mỗi lambda, chạy pipeline và lấy:

* x = avgSkillGap
* y = totalProfit

Vẽ scatter/line chart để minh họa trade-off:

lambda nhỏ → lợi nhuận cao hơn nhưng trận lệch hơn
lambda lớn → cân bằng hơn nhưng lợi nhuận có thể giảm
